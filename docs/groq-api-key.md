# Getting a free Groq API key

The daily challenge, tutor chat, and cloze generation features call
[Groq](https://console.groq.com)'s free tier — Llama 3.3 70B, currently
~14 400 requests/day with a generous rate limit. That's enough to run
Lexico for a single user forever without paying a cent.

Without a key, the app still works end-to-end: the dictionary loop (Lookup
→ Decks → Review) is fully offline, and the LLM-backed views fall back to
deterministic stub output with a visible "no real LLM configured" banner
so you know the feature is degraded.

## 1. Create a Groq account

1. Go to <https://console.groq.com>.
2. Sign in with Google or GitHub. There is no credit card prompt — the
   free tier is genuinely free, no trial.
3. Accept the terms.

## 2. Generate an API key

1. In the console, open **API Keys** in the left sidebar.
2. Click **Create API Key**.
3. Name it `lexico` (or whatever you like) and confirm.
4. Copy the key — it starts with `gsk_...`. You only see it once; paste
   it somewhere safe immediately.

## 3. Wire it into Lexico

### Local development

Create `.streamlit/secrets.toml` at the repo root (it's gitignored):

```toml
GROQ_API_KEY = "gsk_..."
```

Or export it as an environment variable:

```bash
export LEXICO_GROQ_API_KEY="gsk_..."
```

Restart `streamlit run src/lexico/ui/app.py`. The banner on the Challenge
and Tutor pages should disappear.

### Streamlit Community Cloud

Open your deployed app's **Settings → Secrets** editor and add:

```toml
GROQ_API_KEY = "gsk_..."
```

Save. Streamlit reboots the app automatically.

## 4. Verify it works

1. Open the **Daily challenge** page.
2. The "⚠️ No real LLM configured" banner should be gone.
3. Save three words to a deck (or let FSRS queue three due cards), write
   a sentence, click **Grade**.
4. You should see a real score, plain-language feedback, and optionally a
   suggested correction. All generated by Llama 3.3 70B via Groq.

## Rate limits and cost caps

Groq's free tier has daily and per-minute request caps. Lexico enforces
its own per-user and global daily caps on top via `UsageGuardrail`:

| Cap | Default | Override |
|---|---|---|
| Per-user calls per day | 50 | `LEXICO_MAX_LLM_CALLS_PER_USER_PER_DAY` |
| Global calls per day | 500 | `LEXICO_MAX_LLM_CALLS_PER_DAY` |
| Global USD per day | **$0.00** | `LEXICO_DAILY_USD_CAP` |

Groq is charged at $0.00/token on the free tier, so the USD cap never
trips for Groq calls. The cap exists to prevent accidental spend if you
ever add a paid provider like Claude later.

## Troubleshooting

- **Invalid API key** — regenerate on the Groq console; the key only
  shows once, so a lost key needs a new one.
- **Rate limit exceeded** — you've blown through Groq's per-minute or
  per-day limit. Wait a minute; the guardrail will re-allow calls.
- **"Daily LLM budget reached" in the UI** — that's the *Lexico* cap,
  not Groq's. Bump `LEXICO_MAX_LLM_CALLS_PER_USER_PER_DAY` if you're
  testing heavily.
- **Banner still shows with a key set** — confirm the secret name is
  `GROQ_API_KEY` exactly (all caps, underscore, no prefix). The env-var
  form is `LEXICO_GROQ_API_KEY`.
